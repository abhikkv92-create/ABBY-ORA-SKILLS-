# CONTRIBUTING.md - Restricted Contribution Policy

**IMPORTANT**: This repository has RESTRICTED contribution terms. Please read carefully.

## Contribution Policy

### Who Can Contribute

This repository is **NOT open for public contributions**. The following individuals may contribute:

- **Abhinav Verma** (@Abhikkv92) - Primary maintainer

### Why Contributions Are Restricted

1. **Attribution Protection** - To prevent unauthorized removal of developer attribution
2. ** Quality Control** - To maintain skill accuracy and consistency
3. **Legal Compliance** - To ensure all contributions comply with the LICENSE

### If You Want To Contribute

**Option 1: Forking (Not Recommended)**
- You may fork for personal use
- You MUST keep all attribution headers
- You cannot claim as your own work

**Option 2: Feature Requests**
- Open an issue with: `[Feature Request]` prefix
- Describe the Oracle Fusion Cloud feature
- Explain the business use case

**Option 3: Bug Reports**
- Open an issue with: `[Bug Report]` prefix
- Include reproduction steps
- Include Oracle version

### What Will NOT Be Accepted

- PRs removing attribution headers
- PRs re-licensing content
- PRs adding incompatible content
- PRs from unknown sources without consent

### Contribution Process (If Approved)

1. Fork the repository (optional)
2. Create a feature branch: `feature/your-feature`
3. Make your changes
4. Include attribution in changes
5. Submit PR with detailed description
6. Wait for review (may take 1-2 weeks)
