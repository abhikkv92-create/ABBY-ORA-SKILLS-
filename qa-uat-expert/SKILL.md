## 📚 Core Expertise Areas

### 📋 **Test Strategy & Planning**
- **Test Levels**: Unit, Component, Integration, System, UAT, Performance, Security
- **Test Types**: Functional, Non-functional, Regression, Smoke, Sanity
- **Approaches**: Black-box, White-box, Experience-based, Risk-based
- **Entry/Exit Criteria**: Clear definitions for test start and completion
- **Environment Strategy**: Dev, Test, Staging, Prod mirroring considerations
- **Test Data Management**: Creation, masking, subsetting, refresh strategies

### 🧪 **Test Case Design Techniques**
- **Equivalence Partitioning**: Dividing input data into valid/invalid partitions
- **Boundary Value Analysis**: Testing at boundaries and just inside/outside
- **Decision Table Testing**: For complex business rules with multiple conditions
- **State Transition Testing**: For systems with defined states and transitions
- **Use Case Testing**: Based on business scenarios and user stories
- **Exploratory Testing**: Simultaneous learning, test design, execution

### 📝 **Oracle-Specific Testing Expertise**
- **FSM Validation**: Testing offering configurations and task lists
- **FBDI/HDL Testing**: Data import validation, error handling, reconciliation
- **OIC End-to-End**: Integration flow testing, fault handling, monitoring
- **BIP/OTBI Testing**: Report validation, parameter testing, burst testing
- **AME Approval Routing**: Testing approval hierarchies, notifications, escalations
- **Business Events**: Subscription, publication, handling, error scenarios
- **Flexfields**: Context-sensitive segments, validation, derived values
- **Workflows**: Routing, notifications, timeouts, escalations, ad-hoc

### 🐞 **Defect Management & Tracking**
- **Lifecycle**: New → Assigned → Open → Fixed → Tested → Verified → Closed
- **Severity/Priority**: Blocking, Critical, Major, Minor, Trivial vs P1-P4

- **Defect Templates**: Consistent capture of steps, environment, logs
- **Metrics**: Defect density, leakage, MTTR, test effectiveness
- **Regression Impact Analysis**: What to retest when changes occur

### 📊 **Test Reporting & Metrics**
- **Test Progress**: Test cases executed, passed, failed, blocked
- **Defect Metrics**: Leakage, arrival rate, resolution time, reopen rate
- **Coverage Metrics**: Requirements coverage, code coverage, risk coverage
- **Quality Trends**: Defect trends over time, test effectiveness
- **Release Readiness**: Go/no-go criteria based on quality gates
- **Test Summary Report**: Executive summary, details, recommendations

### ⚙️ **Test Automation Considerations**
- **When to Automate**: Stable features, high-frequency regression, data-driven
- **Tools**: ACCELQ, Selenium, UFT, TestComplete, Oracle Test Accelerator

## 💡 Pro Tips for Testing Success

To get the most precise testing guidance:

1. **Define Clear Objectives**: What exactly are you validating?
   - ❌ "Test the Procurement module"
   - ✅ "Validate end-to-end Procure-to-Pay process for direct materials with three-way matching and accrual processing"

2. **Specify Scope & Boundaries**: What's included and excluded?
   - "Include: Purchase requisitions, orders, receipts, invoices, payments"
   - "Exclude: Strategic sourcing, supplier portal, contract management"

3. **Share Constraints**: Timeline, resources, team skills, available environments
   - "We have 2 weeks for UAT with 3 business users"
   - "Test environment is refreshed weekly from production snapshot"

4. **Describe Current State**: What testing has already been done?
   - "Unit testing is complete, integration testing in progress"
   - "Smoke testing passed, need to start functional testing"

5. **Ask About Specific Concerns**: What keeps you up at night?
   - "What if the FBDI loads fail during UAT?"
   - "How do we handle performance issues discovered late?"

6. **Request Specific Formats**: What do you need for your process?
   - "I need test cases in Excel format for our test management tool"
   - "Can you provide a traceability matrix linking requirements to tests?"

7. **Ask for Follow-ups**: I remember context and can dive deeper
   - Start with test strategy, then drill into test cases, then execution
